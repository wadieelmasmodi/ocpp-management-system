# Contributing to Savekar Open-Source CSMS

First off, thank you for considering contributing to Savekar's Open-Source Charge Point Management System! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
Bugs are tracked as GitHub issues. When you create an issue, please use the Bug Report template and provide as much detail as possible:
* Use a clear and descriptive title.
* Describe the exact steps which reproduce the problem.
* Include screenshots and animated GIFs if possible.
* If it's an OCPP-specific bug, please include the relevant OCPP message logs.

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. Please use the Feature Request template to guide you through providing the necessary details.

### Pull Requests
1. Fork the repository and create your branch from `main`.
2. Name your branch appropriately (e.g., `feature/awesome-new-thing` or `fix/button-alignment`).
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes (if applicable).
5. Make sure your code conforms to the existing style (TypeScript, Prettier/ESLint formats).
6. Update the documentation with details of changes to the interface, testing, etc.

## Development Setup
Please refer to the `SETUP.md` file in the root directory for instructions on setting up your local development environment. Both the Frontend (Next.js) and Backend (Node.js/Express) should be tested before submitting a pull request.
